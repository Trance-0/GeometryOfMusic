# 0.0.8

**Date:** 2026-04-20.

Timeline editing + playhead interaction brought closer to the
conventions of mainstream DAWs.

## Left-click places, right-click removes

Timeline cells previously toggled: left-click an empty cell added a
dyad, left-click a filled cell removed it. That ambiguity surprised
users. Fix:

- **Left-click** a cell always places the current dyad (overwriting
  any chord covering that cell).
- **Right-click** a cell removes the placement covering it.
- `contextmenu` is suppressed on timeline cells so the browser menu
  doesn't appear.
- `TimelineCallbacks` gets a new `onCellRemove(trackIndex,
  cellIndex)`. `onCellClick` is now strictly "place current dyad".

Keyboard navigation is unchanged: **Enter** places, **Backspace**
clears, arrow keys move the selection cursor.

## Playhead is grabbable and survives pause / stop

Round 7 shipped a smooth red playhead line but hid it when transport
stopped, so the user couldn't see or interact with it between
sessions. This round makes the playhead a **first-class draggable
control**, like the playhead in GarageBand / Ableton / FL Studio.

- The red line is now visible from app start, parked at cell 0. It
  stays visible through pause, stop, and arbitrary scrubs.
- `.tl-playhead` has `pointer-events: auto` and `cursor: ew-resize`.
  An invisible widened hit area (`::after`) extends the grip
  sideways so the 2 px line is easy to target. A larger round knob
  at the top (`::before`) provides a clear drag handle.
- `TimelineView.installPlayheadDrag` + `startScrubDrag` let the user
  grab the playhead anywhere and drag it across the cells. Matches
  the header-ruler scrub gesture exactly; both end up calling
  `onScrub(cellIndex)`.
- The header ruler's background is tinted and cursor is
  `ew-resize` so users discover the scrub affordance.

## Play = Play/Pause; Stop = pause + rewind

The Play button and Stop button used to do the same thing (both
called `stop()` which cleared the playhead and reset the scheduler).
Split:

- **Play button** toggles Play / Pause. Pause halts the scheduler
  and audio but leaves the playhead where it was. Play resumes
  from `state.playheadIndex`.
- **Stop button** pauses and rewinds the playhead to cell 0.

The playhead remains visible in both cases, so the user can always
see and grab it.

## Behaviour when scrubbing during play

During playback, a scrub cancels the current step timeout, silences
any held notes, and restarts `playStep(cellIndex)` at the new cell.
The audio jumps with the line. This matches what "click-to-jump" does
in a DAW.
