# 0.0.4

**Date:** 2026-04-20.

Two threads in this round: a new minimal-mode toggle on the 3D view,
and a docs reorganization that splits the old `docs/readme.md` into
per-experiment and per-version pages.

## Hide torus

- New **Hide torus** checkbox next to the **Reset view** button in
  the upper-left overlay of the 3D view.
- When checked:
  - The torus wireframe shell is hidden.
  - The voice-leading grid edges are hidden.
  - Every dyad node that is **not** part of a current track curve is
    hidden.
  - The currently highlighted node stays visible even if unused, so
    picking by keyboard / picker is still usable.
- Preference persists in `localStorage` under `gom.hideTorus`.

The intent is a minimal visual mode showing only the three track
curves and the vertices they pass through — useful for reading a
progression without the distraction of the 144-node lattice.

## Docs reorganization

- **Deleted** `docs/readme.md`; its content has been merged into
  `docs/index.md` (landing), `docs/experiment/torus-dyad.md` (the
  torus-specific math / theory / implementation), and
  `docs/versions/` (one page per round for changelog).
- **New `docs/experiment/`** (user-created folder) with one page per
  musical space. Each
  page has three fixed sections: music-theory reading, mathematical
  reading, and how the app renders it (or plans to). Covers:
  `torus-dyad.md` (shipped), `tonnetz.md`, `dyad-orbifold.md`,
  `triad-orbifold.md`, `voice-leading.md`,
  `schoenberg-regions.md`.
- **New `docs/versions/`** with one page per pre-0.1 round
  (0.0.1 through 0.0.4), plus an `index.md` that tabulates them.
- `mkdocs.yml` nav updated with **Experiments** and **Versions**
  sections; old `readme.md` link removed.
