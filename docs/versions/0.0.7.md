# 0.0.7

**Date:** 2026-04-20.

Three corrections on top of round 5's playback visualization work.

## Playhead moved from the torus to the timeline

Round 5 drew a red progress marker and connecting line on the 3D
view. Round 7 removes that entirely — the torus is visually busy
enough without it — and replaces it with a **smooth red vertical
line** on the 2D timeline.

- `TorusView.setPlayhead(dyads)` is gone. So is the sphere pool,
  the connecting-line mesh, and their materials / geometries.
- New `TimelineView.setPlayheadPosition(cellIndex, stepMs)` places
  a `.tl-playhead` element (absolutely positioned inside
  `#timeline-view`) at the left edge of cell `cellIndex` on the
  cells grid. The transition is `transform stepMs ms linear`, so
  the line glides from one cell to the next as the scheduler
  advances; the glow node at the top of the line is a pure CSS
  circle (`::before`).
- Per-cell yellow "active" outline and `TimelineView.setActiveCell`
  are removed; the playhead line replaces them.
- On the loop wrap from the last cell back to cell 0 the line
  snaps without animation (transition = "none") so it doesn't
  race backwards across the entire row.
- Scrub: pointer-down on the header-row's cell container (the beat
  numbers) starts a scrub drag. Moving horizontally sends the
  target cell through `TimelineCallbacks.onScrub(cellIndex)`. If
  transport is paused the playhead just moves and the next Play
  starts there; if playing, the scheduler cancels its current
  timeout and resumes from the scrubbed cell.

## Highlight glow now matches the track color

Round 5 correctly re-tinted the node body with the track color,
but the emissive glow still used the interval-class base color.
Result: a pink-track node's highlight glowed yellow (or whatever
its interval class happened to be). Fix:

- `highlightDyads` now picks the glow color from
  `trackColorMap.get(key) ?? baseColor`.
- `applyTrackNodeColors` keeps emissive in sync with the body color
  at rest, so a re-highlight later starts from the right base.

## Timeline cells are colored per track

Filled cells were `var(--accent-dim)` (blue) across every track —
the same blue for Lead, Bass, and Pad. Fix:

- `renderTrackCells` writes the track's color onto each filled
  cell as the CSS custom property `--cell-color`. The
  `.tl-cell.filled` rule now reads `background: var(--cell-color,
  var(--accent-dim))`, so:
  - row 1 (Lead) = gold,
  - row 2 (Bass) = sky blue,
  - row 3 (Pad) = pink.
- Cell text color switched to dark (`#1a1a1a`, or `#0b0d12` in
  light theme) so the dyad name remains readable against the
  bright track colors.
