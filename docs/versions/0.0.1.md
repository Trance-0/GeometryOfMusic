# 0.0.1

**Date:** 2026-04-20.
**Commit:** `eb659c4`.

First skeleton. Enough surface to demo the idea and verify the
deployment pipeline.

## What shipped

- Vite + strict TypeScript + three.js single-page app.
- **Torus (ordered dyads)** as the only enabled space:
  - 144 dyad nodes on a 3D torus, colored by interval class.
  - Voice-leading edges between semitone neighbours.
  - Single yellow polyline tracing the composed progression.
- **Timeline**: one flat grid of cells, click-to-place / click-to-clear,
  no tracks yet.
- **Transport**: Play / Stop, BPM, meter (3/4, 4/4, 6/8), subdivision
  (1/4, 1/8, 1/16), bars (1–16), current-dyad picker.
- **Audio**: single `DyadSynth` class — one `AudioContext`, triangle
  oscillators, short attack / release envelope.
- **Navbar** with space placeholders for Tonnetz, Möbius dyad
  orbifold, triad orbifold, voice-leading lattice, and Schoenberg
  functions (all disabled stubs).
- **Docs site** (mkdocs-material) with deployment guides for
  Cloudflare Pages, Vercel, and GitHub Pages.
- **CI**: `deploy-app.yml` (Cloudflare Pages) and `deploy-docs.yml`
  (GitHub Pages).
- **Agent contract**: `AGENTS.md` wired as a git submodule to
  `github.com/Trance-0/AGENTS.md`; `CLAUDE.md` at root points to the
  submodule's `AGENTS.md`.

## Known gaps (addressed later)

- No track / instrument selection.
- No chord duration — every chord plays for one step, no held notes.
- Left-click on a node did nothing.
- Torus wireframe and node grid were misaligned (fixed in 0.0.2).
